# 输入资料分类规则

## 四级优先级

### P0 考纲层 — 必读，决定一切
**特征**：
- 文件名含"复习重点"、"考点"、"考纲"、"老师笔记"
- 内容口语化（如老师录音整理）
- 篇幅短（≤20KB），但包含"会考/不考"等明确信号

**典型示例**：
- `复习重点.txt`（老师录音整理）
- `期末考点.md`
- `考前划重点.docx`

**处理方式**：
- 100% 完整读取
- 提取所有"会考"信号 → 考点清单
- 提取所有"不考"信号 → 排除清单
- 章节优先级表

---

### P0 题源层 — 必读，输出物的核心
**特征**：
- 文件名含"真题"、"试卷"、"试题"、"期末"、"年份"
- 含完整题目（选择/填空/简答/计算）
- 通常为 .doc/.docx/.pdf

**典型示例**：
- `2024期末试卷.doc`
- `2021A卷.docx`
- `作业截图.jpg`（截图也是题源）

**处理方式**：
- 每道题独立结构化（YAML 格式）
- 必标字段：题型/分值/章节归属/考点标签/难度
- 跨年统计考点频次

---

### P1 知识层 — 按需调用，不主动展开
**特征**：
- 教材 PDF（厚度通常 >100 页）
- 课程 PPT（数 MB 到数十 MB）
- 内容学术化、结构化

**典型示例**：
- 教材.pdf
- 第N讲.ppt / .pptx

**处理方式**：
- 先提取为可索引文本（markitdown）
- **只调用 P0 题源/考纲指向的具体部分**
- 用页码、图编号、公式编号作为引用锚点
- 严禁全文复述

---

### P2 辅助层 — 验证用
**特征**：
- 大纲图、思维导图、章节摘要

**处理方式**：
- 对照检查章节完整性
- 不直接产出内容

---

## 自动判定流程

```python
def classify(file_path):
    name = file_path.name.lower()
    
    # P0 考纲层
    if any(kw in name for kw in ['复习重点', '考点', '考纲', '老师笔记', 'reviewpoint']):
        return 'P0-syllabus'
    
    # P0 题源层
    if any(kw in name for kw in ['试卷', '试题', '真题', '期末', '考试', 'exam', 'test']):
        if file_path.suffix in ['.doc', '.docx', '.pdf']:
            return 'P0-questions'
    
    # P0 题源（作业截图）
    if file_path.suffix in ['.jpg', '.png', '.jpeg']:
        if '作业' in name or 'homework' in name:
            return 'P0-questions'
    
    # P1 知识层
    if file_path.suffix == '.pdf' and file_path.stat().st_size > 5_000_000:
        return 'P1-knowledge'
    if file_path.suffix in ['.ppt', '.pptx']:
        return 'P1-knowledge'
    
    # 默认 P2
    return 'P2-auxiliary'
```

---

## 边界情况

1. **真题在 PDF 里**：按 P0 题源处理，但需注意 PDF 提取保真度（必要时配合 OCR）
2. **PPT 里有完整例题解答**：仍归 P1，但例题部分要标记为"高优先级"
3. **复习重点是 PPT 形式**：按 P0 考纲处理（标题/优先级判断为主）
4. **无 P0 考纲层**：触发交互——询问用户"老师有划重点吗？请提供，否则只能基于真题统计推断"
5. **无 P0 题源层**：拒绝执行此 skill——突击复习的本质就是真题反推，没有真题等于无锚点

---

## 反例：不要这样分类

- ❌ 把所有 PDF 都当 P0 → 上下文爆炸
- ❌ 跳过 P0 考纲直接做真题 → 失去章节优先级信号
- ❌ 把课本目录页当 P0 → 没有信号量
