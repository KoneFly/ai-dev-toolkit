# Obsidian 渲染兼容规范

## 强制约束

### 1. KaTeX 公式（不是 MathJax）

Obsidian 默认 KaTeX，支持子集 ≠ LaTeX 全集。

**支持**：
- `$ x^2 $` 行内
- `$$ ... $$` 块级
- `\frac` `\sum` `\int` `\sqrt` `\overline` `\hat`
- `\begin{aligned}...\end{aligned}` 对齐
- `\begin{cases}...\end{cases}` 分段函数

**不支持（需替代）**：
- `\cancel{x}` → 改为「（约去 x）」
- `\xrightarrow[]{}` → 用普通箭头 `\to`
- `\bbox` → 删除装饰，用 `\boxed` 替代
- `\begin{align}` → 改用 `\begin{aligned}`
- `\eqref` `\label` → Obsidian 不支持公式编号系统，用文字注明

### 2. 双链 `[[...]]`

**强烈推荐**：
```markdown
见 [[02_例题精讲#例2.7]]
```

**链接路径规则（极易出错，必读）**：

Obsidian `[[...]]` 内**不支持文件系统语法** `../`！它的路径解析以 Vault 根目录为基准。

| 写法 | 能识别吗 | 说明 |
|------|---------|------|
| `[[../01_第1章_MOS器件/01_考点清单]]` | ❌ | `../` 是文件系统语法，Obsidian 直接报"未找到" |
| `[[01_第1章_MOS器件/01_考点清单]]` | ✅ | 从 Vault 根解析（推荐用于跨章引用） |
| `[[01_考点清单]]` | ⚠️ | 若 Vault 内有多个同名文件会歧义 |

**正确做法**：
- **同章内引用**（同目录）→ 直接 `[[02_例题精讲]]`
- **跨章引用**（不同目录）→ `[[01_第1章_MOS器件/02_例题精讲]]`，**绝不加 `../`**
- **指向 Vault 根目录的文件** → `[[复习重点.txt]]`（Vault 根设置由用户决定）

**注意**：
- 章节锚点是原始 heading 文本（含 emoji 也算）
- 跨目录链接需要 Obsidian Vault 内全文件名唯一（建议不重名）
- 链接显示别名：`[[02_例题精讲#例2.7|例2.7]]`

### 3. Callout 块

```markdown
> [!note] 一般提示
> 内容

> [!warning] 警示
> 内容

> [!faq]- 折叠答案（默认收起）
> 答案在这里

> [!faq]+ 折叠答案（默认展开）
> 答案在这里
```

`[!type]` 后接 `-`（折叠）/ `+`（展开）/ 空（不可折叠）。

### 4. Mermaid

**支持类型**：
- `graph LR/TB` 流程图
- `sequenceDiagram` 时序图
- `stateDiagram-v2` 状态机
- `classDiagram` 类图
- `flowchart` 与 `graph` 同义但更新语法

**注意**：
- 中文节点名加引号：`A["第1章"]`
- 颜色用 `classDef ... fill:#xxx`
- 不支持 `mindmap`（部分版本支持，避免依赖）

### 5. 标题层级

- `#` 文件总标题（每文件唯一）
- `##` 主要章节
- `###` 子章节
- 不要跳级（# 后直接 ### 会破坏大纲）

### 6. 不要使用

| 禁用 | 原因 | 替代 |
|------|------|------|
| HTML `<details>` | Obsidian 不解析 | Callout `[!faq]-` |
| HTML `<span style>` | 不渲染 | Markdown 加粗/斜体 |
| `<br>` | 不渲染 | 段落间空一行 |
| emoji | 污染大纲视图 | 文字 |
| Setext 标题 (`===`/`---` 下划线式) | 与水平线冲突 | ATX 标题 (`#`) |

## 图片嵌入规范

### 引用语法

```markdown
![[_figures/fig_电路拓扑_共源放大器.png]]
```

- 使用 Obsidian 内部嵌入语法 `![[...]]`（不用标准 Markdown `![]()`）
- 图片放在**同章节目录**下的 `_figures/` 子目录
- 引用时用相对路径（Obsidian 会自动在 Vault 中搜索同名文件）

### 图片命名规范

```
fig_{类型}_{描述}.png
```

| 类型前缀 | 含义 | 示例 |
|---------|------|------|
| `fig_电路拓扑_` | 原始电路原理图 | `fig_电路拓扑_共源放大器.png` |
| `fig_小信号_` | 小信号等效电路 | `fig_小信号_共栅放大器.png` |
| `fig_特性曲线_` | I-V、频响等曲线 | `fig_特性曲线_MOS三区IV.png` |
| `fig_结构_` | 器件截面/版图 | `fig_结构_NMOS截面.png` |

### 嵌入位置

- `01_考点清单.md`：在需要结构说明的考点后紧跟图片
- `02_例题精讲.md`：在"电路图"或"小信号等效电路"说明处嵌入
- `03_公式手卡.md`：一般不需要图片
- 在图片前后各空一行，确保渲染正确

### 图片质量要求

- DPI：200（阅读清晰且文件小）
- 格式：PNG（无损）
- 单张大小：≤200KB
- 背景：白底（裁剪时注意不要切到公式/标注）

## 可选增强

### 1. dataview 插件

若用户已装 dataview，可在 `00_总索引.md` 加：

```markdown
## 章节自动索引

\`\`\`dataview
TABLE file.cday as 创建日期, file.size as 大小
FROM "ai复习"
WHERE file.name != "00_总索引"
SORT file.name ASC
\`\`\`
```

但**默认不依赖任何插件**——突击场景没时间装插件。

### 2. Canvas (.canvas 文件)

复杂章节关联用 Canvas 图：

```json
{
  "nodes": [
    {"id":"ch2", "type":"file", "file":"02_第2章/01_考点清单.md", "x":0, "y":0, "width":400, "height":300}
  ],
  "edges": [...]
}
```

仅在章节关系图 Mermaid 表达力不足时使用（一般用不到）。

### 3. Project Graph

外部桌面工具，适合**超大图谱**（几十个节点交错）。突击复习几乎用不到，仅作为备选。

## 验证清单

输出前必须自检：

- [ ] 所有 `$...$` 在同一行内（不跨行）
- [ ] 所有 `$$...$$` 单独成行
- [ ] 所有 `[[link]]` 目标文件存在
- [ ] **所有 `[[...]]` 内部不含 `../`**（这是文件系统语法，Obsidian 不识别）
- [ ] Mermaid 块语法正确（中文节点加引号）
- [ ] Callout 类型在 `[note, warning, faq, info, tip, success, question, danger, bug, example, quote]` 内
- [ ] 文件名无空格、无 emoji、无中文标点
- [ ] 没有出现 HTML 标签
- [ ] 标题层级连续不跳级

不通过 → 修改后重写。
